## CORE CALCULATIONS

File: services/allocationService.ts

### 1. Current Allocation
actualPct(holding) = (holding.quantity × holding.priceEUR) / totalPortfolioValue × 100

### 2. Drift
drift(holding) = actualPct - targetPct
- Positive drift → overweight
- Negative drift → underweight

### 3. Rebalancing Threshold
- Default threshold: ±5% absolute drift
- User-configurable in Settings (options: ±3%, ±5%, ±10%)
- Holdings outside threshold are flagged as "needs rebalancing"

### 4. Rebalancing Suggestion (Cash Injection mode)
Used when user is adding new capital (DCA scenario):

For each holding:
  targetValue = newTotalValue × (targetPct / 100)
  currentValue = holding.quantity × holding.priceEUR
  delta = targetValue - currentValue      -- positive = buy, negative = skip
  unitsToBuy = delta / holding.priceEUR

Rules:
- Never suggest selling in DCA mode (only buy or skip)
- If delta < 0 → skip (already overweight, don't add)
- Round unitsToBuy to 2 decimal places (fractional shares supported on T212)

### 5. Rebalancing Suggestion (Full Rebalance mode)
Used when user wants to rebalance existing portfolio:

For each holding:
  targetValue = totalPortfolioValue × (targetPct / 100)
  currentValue = holding.quantity × holding.priceEUR
  delta = targetValue - currentValue
  units = delta / holding.priceEUR        -- positive = buy, negative = sell

Show both buy and sell suggestions.
Add warning: "Selling may trigger capital gains tax. Consult a tax advisor."

## EDGE CASES TO HANDLE

1. Missing price → exclude holding from calculation, show warning
   "CSBGE7 excluded — price unavailable. Update manually."

2. Target allocations don't sum to 100 → block calculation, show error

3. New holding with no target → show as "Untracked" in allocation table
   Prompt user: "Add a target allocation for [TICKER]?"

4. Zero portfolio value (empty portfolio) → skip all calculations,
   show onboarding message instead

5. Single holding portfolio → show allocation as 100%, 
   rebalancing not applicable

## TYPES

interface AllocationRow {
  ticker: string
  targetPct: number
  actualPct: number
  drift: number           -- actualPct - targetPct
  currentValueEUR: number
  status: "ok" | "overweight" | "underweight" | "untracked" | "no_price"
}

interface RebalanceSuggestion {
  ticker: string
  exchange: string
  action: "buy" | "sell" | "skip"
  units: number
  estimatedValueEUR: number
  reason?: string         -- e.g. "overweight, skip in DCA mode"
}

interface RebalanceResult {
  mode: "dca" | "full"
  cashInput: number
  suggestions: RebalanceSuggestion[]
  totalToDeploy: number
  transactionCount: number
  warnings: string[]
}
